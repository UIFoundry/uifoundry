## Props

The Header Menu Button accepts the following props from PayloadCMS:

<TypeTable
  type={{
    alignment: {
      type: "select",
      description:
        "Alignment position within the header layout (left, center, right)",
      default: "left",
    },
    label: {
      type: "text",
      description: "Button text label",
      default: "Sign In",
      required: true,
    },
    href: {
      type: "text",
      description: "Target link URL when not using authentication",
    },
    targetBlank: {
      type: "checkbox",
      description: "Open link in new tab",
      default: false,
      required: true,
    },
    auth: {
      type: "group",
      description: "Authentication configuration for sign-in functionality",
      fields: {
        provider: {
          type: "select",
          description: "OAuth provider for authentication (google, etc.)",
          default: "google",
        },
      },
    },
  }}
/>

## Features

### Authentication Integration

- **Better Auth Integration**: Seamless OAuth provider support
- **Session Management**: Automatic user state detection
- **Social Sign-In**: Google OAuth and other providers
- **User Avatar**: Shows UserAvatar component when authenticated

### Button Modes

#### Authentication Mode

When `auth` configuration is provided:

- Shows sign-in button for unauthenticated users
- Displays UserAvatar for authenticated users
- Handles OAuth provider selection
- Automatic session state management

#### Navigation Mode

When no `auth` configuration:

- Standard link button with configurable href
- Target blank option for external links
- Custom label text

### Flexible Positioning

- **Left Alignment**: Left-aligned with `self-start`
- **Center Alignment**: Center-aligned with `self-center`
- **Right Alignment**: Right-aligned with `self-end`

### Mobile Responsiveness

- Hidden on mobile when `isMobile={true}` prop is set
- Integrates with Custom Header's mobile menu system
- Responsive alignment classes

## Usage in Custom Header

This component works as a nested block within the Custom Header's three-column layout:

```tsx
// Positioned based on alignment prop
<RenderBlocks
  blocks={(items ?? []).filter((i) => i.alignment === "right")}
  blockComponents={blockComponents}
/>
```

Common patterns:

- **Sign In Button**: `alignment="right"` with auth configuration
- **Navigation Button**: Any alignment with href configuration
- **User Profile**: Automatically shows when user is authenticated

## Authentication Example

```tsx
// PayloadCMS configuration for auth button
{
  label: "Sign In",
  alignment: "right",
  auth: {
    provider: "google"
  }
}
```
