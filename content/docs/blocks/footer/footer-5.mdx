## Props

The Footer_5 block accepts the following props from PayloadCMS:

<TypeTable
  type={{
    brandLogo: {
      type: "MediaField",
      required: false,
      description: "Brand logo displayed at the top with theme support (see Media structure below)",
    },
    copyright: {
      type: "CopyrightGroup",
      required: false,
      description: "Copyright information with optional action links in rounded container (see CopyrightGroup structure below)",
    },
    links: {
      type: "Array<LinkGroup>",
      required: false,
      description: "An array of link groups with titles and items for navigation (see LinkGroup structure below)",
    },
    socialLinks: {
      type: "Array<SocialLink>",
      required: true,
      description: "Array of social media links with icons displayed in header section (see SocialLink structure below)",
    },
  }}
/>

### Media

The brand logo is a standard PayloadCMS media object:

<TypeTable
  type={{
    url: {
      type: "string",
      required: true,
      description: "URL path to the logo image",
    },
    alt: {
      type: "string",
      required: true,
      description: "Alt text for the logo image",
    },
  }}
/>

### CopyrightGroup

The copyright section includes text and optional action links:

<TypeTable
  type={{
    copyright: {
      type: "string",
      required: false,
      description: "Copyright text (e.g., 'UIFoundry, All rights reserved')",
      default: "UIFoundry, All rights reserved",
    },
    actions: {
      type: "Array<ActionLink>",
      required: false,
      description: "Array of action links displayed in the copyright section (see ActionLink structure below)",
    },
  }}
/>

### ActionLink

Each action link in the copyright section:

<TypeTable
  type={{
    label: {
      type: "string",
      required: false,
      description: "Display text for the action link",
      default: "License",
    },
    href: {
      type: "string",
      required: false,
      description: "Target URL for the action link",
      default: "/",
    },
  }}
/>

### LinkGroup

Each link group in the `links` array has the following structure:

<TypeTable
  type={{
    group: {
      type: "string",
      description: "Title/heading for the link group (e.g., 'Product', 'Company', 'Legal')",
      required: false,
    },
    items: {
      type: "Array<LinkItem>",
      description: "Array of navigation links within this group",
      required: false,
    },
  }}
/>

### LinkItem

Each navigation link within a group's `items` array:

<TypeTable
  type={{
    label: {
      type: "string",
      description: "Display text for the navigation link",
      required: true,
    },
    href: {
      type: "string",
      description: "Target URL for the navigation link",
      required: true,
    },
  }}
/>

### SocialLink

Each social link in the `socialLinks` array:

<TypeTable
  type={{
    href: {
      type: "string",
      description: "URL for the social media profile",
      required: true,
    },
    icon: {
      type: "SocialIconKey",
      description: "The social media platform icon to display (X, LinkedIn, Facebook, Instagram, TikTok, Threads, etc.)",
      required: true,
    },
  }}
/>

## Installation

```bash
npx shadcn@latest add @uifoundry/footer-5
```

If you encounter issues, check the [registry setup guide](/docs/registry-setup) for configuration help.

## Registry Dependencies

This component uses the following registry components:

- [Style Utils](/docs/lib/style-utils) - Tailwind utility functions including `cn()` class merger
- [Media Field](/docs/fields/media-field) - PayloadCMS dual upload field (light/dark variants)
- [Social Links Field](/docs/fields/social-links-field) - PayloadCMS field for managing social media links with icon selection
- [Block Constants](/docs/lib/block-constants) - Block slug and group constants for PayloadCMS blocks

## Dependencies

This component requires the following NPM packages:

- [PayloadCMS](https://github.com/payloadcms/payload) - Headless CMS and application framework
- [React](https://github.com/facebook/react) - JavaScript library for building user interfaces
- [Next.js](https://github.com/vercel/next.js) - React framework for production
