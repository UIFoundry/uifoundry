---
title: "Render Blocks"
description: "Component for rendering PayloadCMS blocks dynamically with proper typing"
category: "ui"
---

## Preview

```tsx
<RenderBlocks
  blocks={page.blocks}
  blockComponents={{
    hero: HeroComponent,
    header: HeaderComponent,
  }}
  user={user}
/>
```

## Props

The RenderBlocks component accepts the following props:

<TypeTable
  type={{
    blocks: {
      description: "Array of PayloadCMS blocks to render",
      type: "Block[] | unknown[]",
      required: true,
    },
    blockComponents: {
      description: "Mapping of block types to React components",
      type: "Record<string, ElementType>",
      required: true,
    },
    user: {
      description: "PayloadCMS user object for access control",
      type: "PayloadUser | null",
    },
    params: {
      description: "Next.js route parameters",
      type: "PageParams",
    },
    searchParams: {
      description: "Next.js search parameters",
      type: "PageSearchParams",
    },
  }}
/>

## Installation

```bash
npx shadcn@latest add @uifoundry/renderblocks
```

If you encounter issues, check the [registry setup guide](/docs/registry-setup) for configuration help.

## Registry Dependencies

This component has no registry dependencies.

## Dependencies

This component requires the following NPM packages:

- [PayloadCMS](https://github.com/payloadcms/payload) - Headless CMS and application framework
- [React](https://github.com/facebook/react) - JavaScript library for building user interfaces
